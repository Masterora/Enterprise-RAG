Enterprise-RAG 将分散的制度、规范、手册和技术资料整理为统一知识库。用户提问后，Agent 根据问题动态选择知识检索、知识概览、文档定位或联网搜索工具，汇总执行结果并生成带来源的回答。

## 项目特点

| 结构化入库 | Agent 执行 | 可信回答 | 可观测优化 |
| :--- | :--- | :--- | :--- |
| 识别标题、段落、页码和表格，尽量保留文档上下文 | 状态机按观察结果迭代规划，并行执行必要工具 | 只展示真正支撑回答的内部文档或外部网页来源 | Grafana 指标、模型成本与 Jaeger 全链路追踪 |

项目不把 Agent 固化成特定知识库的关键词分支。工具通过统一协议注册，模型只能从当前允许的工具中规划任务；最大工具数、超时、参数长度和观察范围均受配置约束。

Agent 执行状态由显式状态机约束；文档与索引任务状态通过 PostgreSQL 原子条件更新，避免重复消费、乱序消息或删除并发导致状态回退。

模型能力通过统一的 Provider 接口接入。单体后端负责模型选择、客户端复用、超时重试和错误分类，Agent、检索与文档索引复用同一套模型配置。

## 系统架构

![Enterprise-RAG 系统架构](./docs/system-architecture.svg)

系统采用模块化单体架构。Go-zero API 统一承载账号、知识库、文档、检索、Agent 和会话能力；文档处理 Worker 在同一进程中并发消费 NATS 任务，领域逻辑仍按模块隔离，便于后续按真实负载拆分。

## 核心链路

| 文档入库 | Agent 问答 |
| --- | --- |
| 多文件批量上传 | 动态分析目标并选择可用工具 |
| Markdown、TXT、PDF、DOC、DOCX、PPTX 结构化解析 | 知识检索、知识概览、文档定位、联网搜索 |
| 解析、切块、向量化并发处理 | 多工具并行执行，限制超时、数量和参数长度 |
| NATS 任务流转与 Worker 幂等消费 | 规划、执行、观察循环受状态机与预算约束 |
| 失败定位、任务重试和状态追踪 | SSE 流式展示每轮规划、工具与观察步骤 |
| 删除原文件、业务记录和向量索引 | 资料不足时明确回答无法确定 |

```text
文件  →  解析  →  切块  →  向量化  →  建立索引
问题  →  规划  ⇄  工具执行  →  观察  →  回答 + 引用
```

## 快速开始

### 环境要求

| Docker | Go | Node.js | pnpm |
| --- | --- | --- | --- |
| Docker Compose | `1.26+` | `20.19+` 或 `22.12+` | `9+` |

### 启动基础服务

```bash
docker compose up -d
```

基础服务包括 PostgreSQL、NATS、MinIO、Etcd、Milvus，以及 Prometheus、Grafana、OpenTelemetry Collector 和 Jaeger。

| 入口 | 地址 | 用途 |
| --- | --- | --- |
| Grafana | `http://localhost:3000` | 查看预置运行总览、请求趋势、失败率和核心链路耗时 |
| Jaeger | `http://localhost:16686` | 定位单次请求中的 Agent、检索、模型和工具耗时 |
| Prometheus | `http://localhost:9090` | 查询原始指标与告警状态 |

后端启动后会向 OpenTelemetry Collector 上报调用链，并由 Prometheus 抓取 `9999` 端口的运行指标。基础告警规则位于 `deploy/monitoring/alerts.yaml`。

### 启动后端服务

```bash
export OPENROUTER_API_KEY=你的密钥
cd backend
go run .
```

后端默认监听 `http://localhost:9999`，并在同一进程中启动 API 与文档任务 Worker。

### 启动前端

```bash
cd frontend
pnpm install
pnpm dev
```

前端默认地址：`http://localhost:5173`

## 使用流程

1. 注册并登录系统。
2. 创建知识库并上传文档。
3. 等待文档状态变为“已索引”。
4. 进入助手页面，选择知识库和模型后提问。
5. 阅读回答，并通过引用来源核查原文。
6. 在日志页面查看处理状态；在检索评估中验证召回效果。

## 质量边界

- 扫描图片、复杂图表、加密文件和低质量文档可能影响解析精度。
- 联网资料只作为知识库之外的补充，不能替代企业内部正式依据。
- 单题 Recall@K 用于验证指定问题的召回结果，不代表模型回答在所有问题上都正确。
- 法律、医疗、财务和生产控制等高风险结论仍需专业人员确认。
